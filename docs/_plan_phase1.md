## 1. 기존 문서에서 이미 정의된 내용

### `docs/systems/overview.md` — 보드 시스템 섹션

아래 항목은 `overview.md`에 명확히 서술되어 있어 재작성 불필요.

| 항목 | 정의 내용 |
|------|-----------|
| 보드 기본 크기 | 4×4 (16칸). 유물·패시브에 의해 확장 가능 |
| 슬라이드 동작 | 지정 방향으로 모든 타일이 밀림. 인접한 동일 숫자 타일은 병합되어 두 배 값이 됨 |
| 새 타일 생성 타이밍 | 슬라이드 1회 후 빈 칸 중 무작위 1칸에 생성 |
| 새 타일 후보 값 | 2 또는 4 |
| 슬라이드 횟수 제한 | 턴당 제한 존재 (기본값 미정) |
| 타일 소모 조건 | 카드 발동 숫자와 정확히 일치해야 소모. 상위 숫자 대체 불가 |
| 행동 불능 조건 | 보드가 꽉 차서 슬라이드해도 어떤 타일도 이동하지 않는 경우 |
| 행동 불능 효과 | 슬라이드와 카드 발동 모두 차단 |
| 유효 슬라이드 기준 | 병합이 없더라도 타일이 물리적으로 이동했으면 유효. 카운트 소모 + 새 타일 생성 발생 |

### `docs/index.md` — 전투 시스템 보드 관련 내용

| 항목 | 정의 내용 |
|------|-----------|
| 보드 위기 상황 | 보드가 꽉 차면 슬라이드 시 새 타일이 생성되지 않음 (overview.md와 일부 상충 — 아래 미결 사항 참고) |
| 슬라이드의 적 카운트 영향 | 슬라이드는 적 행동 카운트에 영향을 주지 않음 |

## 2. 1단계 구현 항목별 문서화 상태

| 구현 항목 | 문서화 상태 | 비고 |

| 턴당 슬라이드 횟수 제한 처리 | 규칙 언급만 | 제한 존재 확인됨. **기본 횟수값, 소진 시 동작(슬라이드 입력 차단 여부), 횟수 리셋 타이밍 미정** |
| 행동 불능 판정 | 조건·효과 정의 | 조건과 효과 모두 명시됨. **판정 시점(슬라이드 시도 시점인지 슬라이드 횟수 소진 후인지), 해제 조건 미정** |

### 3-E. 문서 간 기술 불일치 — 확인 필요

`index.md`와 `overview.md` 사이에 아래 표현이 상충한다. 구현 전 하나로 통일해야 한다.

| 항목 | `index.md` 기술 | `overview.md` 기술 |
|------|-----------------|--------------------|
| 보드가 꽉 찼을 때 슬라이드 결과 | "새 타일이 생성되지 않음" (슬라이드 자체는 가능) | "슬라이드해도 어떤 타일도 이동하지 않으면 행동 불능" (조건이 다름) |

→ 실제로 두 상황은 별개다: (상황 A) 보드가 꽉 찼지만 병합이 일어나는 경우 / (상황 B) 어느 방향으로도 타일 이동이 없는 경우. 두 상황을 구분하여 각각의 동작을 명시해야 한다.

## 4. 작성이 필요한 문서와 섹션 계획

### 우선순위 높음 — 1단계 구현에 직접 필요

#### 4-1. `docs/systems/board.md` 신규 작성 (최우선)

기존 `overview.md` 보드 섹션의 규칙을 포함하되, 구현자가 코드 작성에 바로 참조할 수 있는 수준으로 상세화한다.

작성할 섹션:

```
1. 보드 데이터 구조
   - 4×4 2차원 배열 표현 방식
   - 타일 단위 데이터 (값, 위치, 이번 슬라이드에서 병합 여부 플래그)
   - 빈 칸 표현 방식 (null / 0 / 별도 타입)

2. 슬라이드 알고리즘 명세
   - 4방향별 처리 순서 정의 (예: 왼쪽 슬라이드 시 각 행을 왼쪽부터 순회)
   - 타일 이동 로직 (빈 칸 채우기 순서)
   - 병합 로직 (같은 값 감지 → 병합 → 병합 완료 플래그 설정)
   - 연쇄 병합 허용 여부 명시 [미결 #8 확정 후 작성]
   - 유효 슬라이드 판정 로직 (이동 전후 보드 상태 비교)

3. 새 타일 생성 규칙
   - 생성 트리거: 유효 슬라이드 완료 직후
   - 생성 위치: 빈 칸 목록에서 무작위 1칸
   - 생성 값: 2 또는 4 (확률 비율 [미결 #5 확정 후 작성])
   - 보드 만원 상태에서의 처리 [미결 #7 확정 후 작성]

4. 슬라이드 횟수 제한
   - 기본 횟수 값 [미결 #1 확정 후 작성]
   - 횟수 카운터 감소 타이밍 (유효 슬라이드 시에만 감소 vs. 시도마다 감소)
   - 횟수 소진 시 입력 처리 [미결 #2 확정 후 작성]
   - 횟수 리셋 타이밍 [미결 #3 확정 후 작성]

5. 행동 불능 판정
   - 판정 조건: 4방향 모두에서 유효 슬라이드가 불가능한 상태
   - 판정 시점 [미결 #10 확정 후 작성]
   - 행동 불능 상태의 효과 (슬라이드 입력 차단, 카드 발동 차단)
   - 해제 조건 [미결 #11 확정 후 작성]
   - 행동 불능 시 적 행동 처리 [미결 #12 확정 후 작성]

6. 전투 시작 시 보드 초기화
   - 초기 보드 상태 [미결 #6 확정 후 작성]
   - 초기화 순서 (빈 보드 → 초기 타일 배치 → 핸드 드로우와의 순서 관계)

7. 예외 케이스 목록
   - index.md / overview.md 불일치 항목 통합 정리 [미결 3-E 확정 후 작성]
   - 보드 꽉 참 + 슬라이드 시 병합 발생 케이스
   - 보드 꽉 참 + 슬라이드 시 병합도 없는 케이스 (행동 불능)
```

#### 4-2. `docs/schema.md` — 보드 관련 데이터 구조 섹션 추가

현재 완전 플레이스홀더 상태. 1단계 구현에 필요한 범위만 우선 작성한다.

작성할 섹션:

```
BoardState
  - board: number[][] (4×4, 빈 칸은 0)
  - slideCount: number (현재 턴 남은 슬라이드 횟수)
  - maxSlides: number (턴당 최대 슬라이드 횟수)
  - isActionLocked: boolean (행동 불능 여부)

Tile (슬라이드 연산 중 임시 구조)
  - value: number
  - mergedThisTurn: boolean (이번 슬라이드에서 병합된 타일인지)
```
